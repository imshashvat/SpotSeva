01 Project Overview & Problem Analysis

Adopt-a-Spot is a gamified civic maintenance mobile application built on Flutter + Firebase. Citizens "adopt" public micro-spots (a bench, a bus stop, a park entrance), perform digital check-ins, report issues with photos, and earn redeemable reward points. Municipal bodies gain a real-time, crowdsourced map of urban micro-issues — dramatically cutting complaint-to-fix response time.

Core Problem: Municipal corporations remain unaware of micro-degradation events (broken bench, fallen tree, blocked drain) until formal complaints trickle in — often weeks later. Adopt-a-Spot converts every citizen who passes a public spot into a real-time sensor.

02 System Architecture Overview

The platform uses a layered, event-driven architecture with three actor tiers: Citizens, Municipal Dashboard, and Firebase Cloud. All state flows upward through Firestore real-time listeners; background logic executes in Cloud Functions triggered by database events or HTTP calls.

03 Backend Architecture Deep-Dive

The backend is serverless-first: Firebase acts as the infrastructure layer, Cloud Functions handle all business logic, and Firestore security rules act as the API gateway. This eliminates server management, scales infinitely, and keeps costs near-zero at indie scale.

Fig 2. Cloud Functions Trigger Pipeline — six event-driven functions covering the full citizen lifecycle.

Firebase Security Rules Strategy

// Firestore Security Rules (simplified)
rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {

    // Only authenticated users can read spots
    match /spots/{spotId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == resource.data.adopter_id;
    }

    // Reports: create by any logged-in user, update only by municipal role
    match /reports/{reportId} {
      allow create: if request.auth != null;
      allow update: if request.auth.token.role == 'municipal';
    }

    // Points wallet: read-only for user, write only via Cloud Functions
    match /wallets/{userId} {
      allow read: if request.auth.uid == userId;
      allow write: if false; // Functions only
    }
  }
}

07 Feature Modules

🗺️ Interactive Spot Map

Google Maps SDK with clustered spot markers. Color-coded by status (green = clean, amber = issue reported, red = high priority). Tap any spot to see adoption status, last check-in time, and open reports. Geofence alerts when entering 100m of your adopted spot.

📌 Spot Adoption Flow

Search or tap to adopt a free spot. One spot per citizen at a time (promotes fairness). 7-day cooldown after release. Adopter receives push notification if someone else reports an issue at their spot before them.

📸 Issue Report

Tap "Report Issue" → Camera opens → Photo + auto-GPS stamp → Select category (Broken, Dirty, Overgrown, Safety Hazard, Other) → Submit. Gemini Flash auto-classifies severity in under 2 seconds. Citizen gets +25 points immediately.

✅ Check-in System

GPS proximity check (must be within 100m). Daily limit of 2 check-ins per spot to prevent gaming. +10 points per check-in. Streak bonuses: 7-day streak = 50 bonus pts, 30-day = 200 bonus pts. Streak counter on profile screen.

🏆 Leaderboard

Weekly, monthly, and all-time tabs. Ranks by total points in period. Top 3 get a special frame on their avatar. Opt-out option for privacy. Cloud Function recomputes rankings every hour using Firestore batch writes.

🎁 Rewards Store

Business partners upload coupons (e.g. "20% off at Chai Wala, Sector 18"). Citizens browse by category, redeem points atomically. QR code generated for in-store scanning. Coupon expiry enforced server-side.
